# Lens вЂ” BMAD Module Release

This directory is the installed BMAD workspace for the **Lens** module. It contains the full BMad Method installation alongside the Lens lifecycle governance module.

## What Is Lens?

Lens is an AI-guided lifecycle orchestration system built on top of the BMad Method. It treats git as a state machine for AI-assisted software development, providing:

- **Feature-first branching** вЂ” a 2-branch topology (`{featureId}` + `{featureId}-plan`) with automated governance metadata
- **Streamlined planning** вЂ” end-to-end planning pipeline from business plan through story creation in a single flow
- **Retrospective feedback loop** вЂ” problem logging, pattern analysis, and insight curation that compounds across features
- **Progressive disclosure** вЂ” a single `@lens` agent that routes to the right workflow based on lifecycle state

## Installation

The `.github/` adapter layer is auto-generated by the Lens module installer. To regenerate adapters after changes:

```bash
# From this directory
npx bmad-method install
# Select: Modify BMAD Installation в†’ Add custom module в†’ path to dist/bmad-lens-module
```

The installer will:
1. Write `_bmad/lens/bmadconfig.yaml` with runtime configuration
2. Generate 21 skill stubs in `.github/skills/bmad-lens-*/SKILL.md`
3. Generate 21 prompt stubs in `.github/prompts/lens-*.prompt.md`

All Lens skills are invoked via the `@lens` agent. Available skills:

| Skill | Purpose |
|-------|---------|

## Adapter Pattern

The `.github/` directory contains thin stubs вЂ” not full skill content. Each stub redirects the IDE agent to load the canonical file from `_bmad/lens/skills/` or `_bmad/lens/prompts/`. This keeps the adapter surface lightweight and ensures all logic lives in the versioned module source.

Example skill stub (`.github/skills/bmad-lens-init-feature/SKILL.md`):
```markdown
---
name: bmad-lens-init-feature
description: 'Feature initialization orchestrator...'
---

> **This is a stub.** Load and execute the canonical skill.

\`\`\`
Read and follow all instructions in: _bmad/lens/skills/bmad-lens-init-feature/SKILL.md
\`\`\`
```

## Configuration

Runtime config is stored in `_bmad/lens/bmadconfig.yaml`. Key settings:

| Key | Description |
|-----|------------|
| `default_track` | Planning track: `full` or `quickplan` |
| `theme` | Persona overlay: `default` or a theme name |
| `activation_mode` | `auto` (load context on activation) or `manual` |
| `target_repos` | Comma-separated list of target repository paths |
| `governance_repo` | Path to the governance repository |
