# Copilot Adapter — lens-work v4

This directory contains the thin VS Code Copilot adapter that wires the `@lens` agent into the IDE. It references the lens-work module by path and does not duplicate module logic.

## Structure

```text
.github/
├── copilot-instructions.md
├── lens-work-instructions.md
├── agents/
│   └── bmad-agent-lens-work-lens.agent.md
├── prompts/
│   ├── lens-onboard.prompt.md
│   ├── lens-preflight.prompt.md
│   ├── lens-init-feature.prompt.md
│   ├── lens-preplan.prompt.md
│   ├── lens-businessplan.prompt.md
│   ├── lens-techplan.prompt.md
│   ├── lens-finalizeplan.prompt.md
│   ├── lens-expressplan.prompt.md
│   ├── lens-dev.prompt.md
│   ├── lens-complete.prompt.md
│   ├── lens-discover.prompt.md
│   └── ... (one `lens-*.prompt.md` stub per published command)
└── skills/
    └── ... (redirect stubs for published BMAD skills and Lens wrappers)
```

## Principles

- **Thin adapter** — references only, never copies or duplicates module content.
- **Path references** — prompts, agents, and skills point back to `lens.core/_bmad/lens-work/`.
- **Auto-propagation** — module updates flow into the adapter through the installer and release pipeline.
- **Generated surface** — `.github/prompts/` contains only the published `lens-*.prompt.md` stub family.

## Generated vs Authored Files

All `.github/` adapter files are generated by the CI/CD pipeline through `_module-installer/installer.js` and the standalone installer in `scripts/install.py`.

- Source of truth for prompt publication lives in `_bmad/lens-work/module.yaml`.
- Source of truth for GitHub prompt stub generation lives in `_bmad/lens-work/_module-installer/installer.js` and `_bmad/lens-work/scripts/install.py`.
- Source prompt content lives in `_bmad/lens-work/prompts/`.

Manual edits to checked-in source stubs under `.github/` are preserved only where the release workflow intentionally overlays authored files on top of generated output.
