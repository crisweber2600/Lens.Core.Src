# Enforcement Workflow Template

_These templates are used by the `generate-workflows` capability to produce GitHub Actions enforcement workflow files. They live in the agent sanctum at `github-templates/workflows.md`. Copy and substitute during generation._

---

## Lens Core Validation Workflow

Use for workspace compliance checking on push or pull request.

```yaml
# lens-core-validate.yml
# ─────────────────────────────────────────────────────────────────────────────
# ENFORCEMENT WORKFLOW — Lens Core Workspace Validation
# Validates workspace structure against Lens Core rules on every PR and push.
#
# Generated by: Governance Enforcement Engine (lc-agent-core-repo)
# Generated on: {generation-date}
# Governance mode: {governance-mode}
# ─────────────────────────────────────────────────────────────────────────────

name: "Lens Core — Workspace Validation"

on:
  pull_request:
    branches: {target-branches}
  push:
    branches: {target-branches}

jobs:
  validate-workspace:
    name: Validate Workspace Structure
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout workspace
        uses: actions/checkout@v4
        with:
          submodules: recursive
          fetch-depth: 0

      - name: Validate required top-level structure
        id: validate-structure
        run: |
          echo "## Lens Core Validation" >> $GITHUB_STEP_SUMMARY
          echo "" >> $GITHUB_STEP_SUMMARY
          
          FAILED=0
          
          # R001: Control Repo Root Structure
          for dir in _bmad src .github; do
            if [ -d "$dir" ]; then
              echo "✅ R001 PASS: $dir/ exists" >> $GITHUB_STEP_SUMMARY
            else
              echo "❌ R001 FAIL: $dir/ missing (required by Lens Core R001)" >> $GITHUB_STEP_SUMMARY
              FAILED=$((FAILED + 1))
            fi
          done
          
          # R002: BMad Config Directory
          if [ -f "_bmad/config.yaml" ]; then
            echo "✅ R002 PASS: _bmad/config.yaml exists" >> $GITHUB_STEP_SUMMARY
          else
            echo "❌ R002 FAIL: _bmad/config.yaml missing (required by Lens Core R002)" >> $GITHUB_STEP_SUMMARY
            FAILED=$((FAILED + 1))
          fi
          
          if [ -f "_bmad/_config/manifest.yaml" ]; then
            echo "✅ R002 PASS: _bmad/_config/manifest.yaml exists" >> $GITHUB_STEP_SUMMARY
          else
            echo "⚠️  R002 WARN: _bmad/_config/manifest.yaml missing" >> $GITHUB_STEP_SUMMARY
          fi
          
          # R003: Source Submodule Convention
          SRC_DIRS=$(find src -maxdepth 1 -name "*.src" -type d 2>/dev/null | wc -l)
          if [ "$SRC_DIRS" -gt 0 ]; then
            echo "✅ R003 PASS: src/*.src/ directories found ($SRC_DIRS)" >> $GITHUB_STEP_SUMMARY
          else
            echo "⚠️  R003 WARN: No src/*.src/ subdirectories found" >> $GITHUB_STEP_SUMMARY
          fi
          
          # R004: GitHub Redirect Stubs — check no implementation files in .github/workflows/
          IMPL_COUNT=$(find .github/workflows -name "*.yml" -exec grep -L "REDIRECT STUB\|delegate\|uses:" {} \; 2>/dev/null | wc -l)
          if [ "$IMPL_COUNT" -eq 0 ]; then
            echo "✅ R004 PASS: .github/workflows/ contains only redirect stubs" >> $GITHUB_STEP_SUMMARY
          else
            echo "⚠️  R004 WARN: .github/workflows/ may contain implementation files (expected redirect stubs only)" >> $GITHUB_STEP_SUMMARY
          fi
          
          echo "" >> $GITHUB_STEP_SUMMARY
          echo "**Total blockers: $FAILED**" >> $GITHUB_STEP_SUMMARY
          
          {fail-behavior}

      - name: Upload validation report
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: lens-core-validation-report
          path: $GITHUB_STEP_SUMMARY
          retention-days: 30
```

**Substitution fields:**
- `{generation-date}` — YYYY-MM-DD
- `{governance-mode}` — `strict`, `advisory`, or `audit-only`
- `{target-branches}` — YAML list, e.g. `[main, 'release/**']`
- `{fail-behavior}` — one of:
  - **strict:** `if [ "$FAILED" -gt 0 ]; then exit 1; fi`
  - **advisory:** `if [ "$FAILED" -gt 0 ]; then echo "::warning::$FAILED Lens Core rule(s) failed — see summary for details"; fi`
  - **audit-only:** `echo "Audit complete. $FAILED rule(s) failed."`

## Adding Custom Rule Checks

To extend the workflow with rules from `rules-extension-points.md`, add steps following this pattern:

```yaml
      - name: Validate extension rule — {extension-name}
        run: |
          # {extension-name}: {rule description}
          # Registered by: {agent-code} on {registration-date}
          
          if {rule-check-command}; then
            echo "✅ {extension-name} PASS" >> $GITHUB_STEP_SUMMARY
          else
            echo "❌ {extension-name} FAIL: {remediation-message}" >> $GITHUB_STEP_SUMMARY
            {fail-behavior}
          fi
```

## Validation Checklist for Generated Workflows

Before writing any workflow to disk, verify:

- [ ] `on:` block targets the correct branches
- [ ] `{fail-behavior}` is consistent with BOND.md governance mode
- [ ] All `find` and `ls` commands have `2>/dev/null` guards for missing directories
- [ ] `GITHUB_STEP_SUMMARY` is used (not `echo` to stdout) for GitHub UI visibility
- [ ] No secrets or tokens referenced directly in the workflow
- [ ] Running this workflow twice would be idempotent (no side effects beyond the summary)
